Git Cheat Sheet

master is the default development branch

origin is the default upstream repository

Create

From existing data:

  1. cd -/my_project_directory
  2. git init
  3. git add .

    4. From existing repository:

    1. git clone -/existing_repo -/new/repo
    2. git clone git://host.org/myproject.git
    3. git clone ssh://user@host.org/project.git

Show

Files changed in working directory

  1. git status

Changes made to tracked files

  1. git diff

What changed between saved version of a file (i.e. ID1 and ID2)

  1. git diff <ID1> <ID2>

History of changes

  1. git log

History of changes for file with diffs

  1. git log -p <FILE> <DIRECTORY>

Who changed WHAT and WHEN with a saved file

  1. git blame <FILE>

A commit identified by ID

  1. git show <ID>

A specific file from a specific ID

  1. git show <ID>:<FILE>

All local branches

  1. git branch

Revert

Return to the last commited state

  1. git reset --hard

Revert to the last command

  1. git revert head

Revert specific commit

  1. git revert <ID>

Fix the last commit

  1. git commit -a amend

Check out the ID version of a file

  1. git checkout <ID> <FILE>

Update

Fetch latest changes from origin

  1. git fetch (this does not merge them)

Pull latest changes form origin

  1. git pull (does a fecth followed by a merge)

Apply a patch that someone sent you

  1. git am -3 patch.mbox (in case of conflict, resolve the conflict then the following...)
  2. git am --resolved

Publish

Commit all your local changes

  1. git commit -a

Prepare a patch for other developers

  1. git format-patch origin

Push changes to origin

  1. git push

Make a version or milestone

  1. git tag v.1.0

Branch

Switch to a branch

  1. git checkout <BRANCH>

Merge BRANCH1 into BRANCH2

  1. git checkout <BRANCH2>
  2. git merge <BRANCH1>

Create branch BRANCH based on head

  1. git branch <BRANCH>

Create branch BRANCH based on OTHER and switch to it

  1. git checkout -b <BRANCH> <OTHER>

Delete branch BRANCH

  1. git branch -d <BRANCH>

Resolve Merge Conflicts

View merge conflicts

  1. git diff

View merge conflicts against base file

  1. git diff -base <FILE>

View merge conflicts against your changes

  1. git diff -ours <FILE>

View merge conflicts against other changes

  1. git diff -theirs <FILE>

Discard a conflicting patch

  1. git reset --hard
  2. git rebase --skip

After resolving conflicts, merge with

  1. git add <CONFLICTING FILE>
  2. git rebase --continue

BREAK IN CASE OF EMERGENCY!

Global configuration is stored in -/.gitconfig

  1. git <COMMAND> --help

Git Help

  1. git config --help
